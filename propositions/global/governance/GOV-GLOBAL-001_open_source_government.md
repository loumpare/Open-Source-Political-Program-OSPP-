## Summary

Any software developed with public funds should be published as open source by default.

## Proposal

Governments at all levels that commission or develop software using public funds must:

1. Release the source code under an OSI-approved license within 90 days of deployment
2. Maintain a public registry of all government software (name, license, repo URL)
3. Prioritize open source solutions in procurement when functionality is equivalent
4. Exception: national security systems (audited annually by independent body)

## Estimated impact

- Applicable to: any democratic government adopting this policy
- Estimated savings: 15-25% on duplicated software development costs (EU Commission, 2024)
- Side effect: creates a public commons of civic software

## Arguments for

- Citizens fund the software — they should have access to it
- Reduces vendor lock-in
- Enables peer review of algorithmic decisions (social scoring, tax calculation, etc.)
- Already adopted partially: Germany, France (Etalab), Estonia

## Arguments against

- Security concerns if vulnerabilities are publicly visible
- Maintenance burden on agencies
- Low-quality code may damage trust
